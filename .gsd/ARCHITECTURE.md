## Data Flow

1. **Upload & Ingest:** User uploads an APK through REST API/Web UI. The system computes SHA-256, stores the file, logs a tamper-evident audit event, and creates a queued Job in the database.
2. **Task Execution:** A Celery background worker picks up the job.
3. **Static Analysis & Feature Extraction:** The worker executes MobSF scan and runs local adapters (Androguard, APKiD, Quark, YARA, certificate parsing). It writes features into a canonical schema.
4. **Local LLM Interpretation:** The worker invokes the GenAI service. APK-derived strings are isolated to prevent injection. The local LLM generates a behavior description and maps it to MITRE ATT&CK.
5. **Grounding & Scoring:** The fusion layer verifies GenAI claims against actual static features. The deterministic rule engine evaluates weights to produce a 0-100 score, mapping it to verdict bands (Benign/Suspicious/Malicious).
6. **Reporting & Sign-off:** PDF/JSON reports are generated and persisted. High/Critical cases are flagged for Analyst sign-off. STIX-2.1 export packages are generated.
